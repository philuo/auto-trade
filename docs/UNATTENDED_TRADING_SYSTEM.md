## 系统目标

创建一个**24/7无人值守、全年无休**的量化交易系统：
- **安全第一**：多层风险控制，防止爆仓
- **稳定运行**：自动故障恢复，断线重连
- **双策略组合**：现货稳健 + 合约进攻
- **数据驱动**：基于统计验证，不依赖主观判断

## 策略分配

### 资金配置（10,000 USDT示例）

| 资金分配 | 金额 | 策略 | 风险等级 |
|---------|------|------|----------|
| 现货DCA-Grid | 6,000 | 稳健积累 | 低 |
| 合约技术分析 | 2,000 | 高胜率交易 | 中 |
| 应急储备 | 2,000 | 不参与交易 | 无 |

### 现货策略（DCA-Grid）

| 特性 | 配置 |
|------|------|
| **目标** | 长期积累筹码，稳健增值 |
| **币种** | BNB, SOL, XRP, ADA, DOGE |
| **模式** | 纯现货，无杠杆 |
| **策略** | DCA定投 + 网格交易 |
| **更新频率** | 5秒 |
| **预期年化** | 30-50% |
| **最大回撤** | 20% |

**运作方式：**
```
下跌 → DCA逆向买入（越跌越买）
震荡 → 网格高抛低吸
上涨 → 网格逐步卖出
```

### 合约策略（技术分析）

| 特性 | 配置 |
|------|------|
| **目标** | 高胜率短线交易，增厚收益 |
| **币种** | BTC, ETH |
| **模式** | 永续合约 |
| **杠杆** | BTC 2x / ETH 2x |
| **策略** | 事件驱动 + ADX过滤 + 统计验证 |
| **更新频率** | 15秒 |
| **最小置信度** | 65% |
| **预期胜率** | 55-65% |
| **最大回撤** | 15% |

**信号示例：**
```
MA7上穿MA25 → 金叉信号 → 检查ADX
若ADX > 25 → 查询历史统计
若胜率 > 50% → 计算置信度
若置信度 > 65% → 执行买入
```

## 风险控制

### 三层防护

```
┌───────────────────────────────────────────────────────────┐
│ 第一层：策略级风险控制                                      │
├───────────────────────────────────────────────────────────┤
│ • 现货：无杠杆，永远不会爆仓                               │
│ • 合约：最大2x杠杆，单笔止损2%                             │
│ • 信号置信度：最低65%才执行                                │
└───────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────┐
│ 第二层：组合级风险控制                                      │
├───────────────────────────────────────────────────────────┤
│ • 现货占总资金60%（安全垫）                               │
│ • 合约占总资金20%（严格限制）                             │
│ • 应急储备20%（绝对不动用）                               │
│ • 总回撤超20% → 暂停新开仓                                 │
└───────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────┐
│ 第三层：系统级风险控制                                      │
├───────────────────────────────────────────────────────────┤
│ • 极端行情检测 → 自动暂停                                  │
│ • 网络断线 → 自动重连                                     │
│ • API限流 → 等待重试                                      │
│ • 异常订单 → 自动撤销                                      │
└───────────────────────────────────────────────────────────┘
```

### 熔断条件

| 条件 | 阈值 | 动作 |
|------|------|------|
| 单K线涨跌 | > ±10% | 暂停新开仓 |
| 1小时波动率 | > 50% | 暂停新开仓 |
| 合约单笔亏损 | > 2% | 立即止损 |
| 总回撤 | > 15% | 警告，降低仓位 |
| 总回撤 | > 20% | 暂停所有交易 |
| 网络断线 | 30秒 | 重连3次，失败则暂停 |

## 24/7运行保障

### 健康检查

| 检查项 | 频率 | 失败处理 |
|--------|------|----------|
| API连接 | 每30秒 | 重连3次 |
| WebSocket | 每10秒 | 自动重连 |
| 持仓状态 | 每5秒 | 检测异常 |
| 未完成订单 | 每30秒 | 检测超时 |
| 系统日志 | 实时 | 异常告警 |

### 故障恢复

```
网络断线
    ↓
检测到断线（10秒内无数据）
    ↓
尝试重连（最多3次）
    ↓
成功 → 继续运行
失败 → 暂停策略，发送告警
    ↓
等待人工干预
```

### 日志记录

| 日志类型 | 记录内容 |
|---------|---------|
| 交易日志 | 每笔交易的完整信息 |
| 性能日志 | 胜率、回撤、盈亏比 |
| 系统日志 | 启动、停止、错误 |
| 告警日志 | 熔断、断线、异常 |

## 启动与运行

### 启动命令

```bash
# 启动完整系统（现货 + 合约）
bun run src/main/unattended-trading.ts

# 仅启动现货策略
bun run src/strategies/spot-dca-grid/demo.ts

# 仅启动合约策略（需单独配置市场数据提供者）
# 技术分析策略位于 src/trading/technical-coordinator.ts
```

### 配置文件

```typescript
{
  // 现货配置
  spot: {
    enabled: true,
    capital: 6000,
    coins: ['BNB', 'SOL', 'XRP', 'ADA', 'DOGE'],
    strategy: 'DCA-Grid'
  },

  // 合约配置
  perpetual: {
    enabled: true,
    capital: 2000,
    coins: ['BTC', 'ETH'],
    leverage: { BTC: 2, ETH: 2 },
    minConfidence: 0.65,
    strategy: 'TechnicalAnalysis'
  },

  // 风险配置
  risk: {
    emergencyReserve: 2000,
    maxDrawdown: 0.20,
    circuitBreakers: {
      priceChange: 10,
      volatility: 50
    }
  }
}
```
